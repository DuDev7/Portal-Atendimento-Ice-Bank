# Portal-Atendimento-Ice-Bank
Projeto acadêmico de portal de atendimento ao cliente desenvolvido com HTML, CSS, JavaScript e PHP, contendo base de conhecimento, FAQ dinâmico, abertura de tickets e área do colaborador com login para visualização e tratamento de chamados.
Projeto acadêmico desenvolvido com o objetivo de simular um portal de atendimento ao cliente de uma instituição bancária, reunindo funcionalidades voltadas tanto ao público externo quanto à área interna de colaboradores.
O sistema foi construído utilizando HTML, CSS, JavaScript e PHP, com apoio de arquivos JSON para armazenamento de dados de forma didática. A proposta do projeto foi aproximar o desenvolvimento acadêmico de um cenário real de suporte e atendimento, organizando recursos úteis para clientes e funcionários do banco.

**Objetivo do projeto**
O principal objetivo deste projeto foi criar uma aplicação web que simulasse o fluxo de atendimento de um banco, permitindo:

Acesso a informações de apoio ao cliente;
Consulta a perguntas frequentes;
Abertura de tickets de atendimento;
Acesso à área do colaborador;
Visualização e tratamento de chamados internos.

Além da proposta acadêmica, o projeto também contribuiu para a prática de organização de páginas web.


**Funcionalidades implementadas**

 Área do cliente
- Página inicial do portal
- Base de conhecimento com artigos de apoio
- FAQ dinâmico com interação em JavaScript
- Abertura de tickets de atendimento
- Sugestão de conteúdo com base no assunto informado


 Área do colaborador
- Tela de login para colaborador
- Painel interno de atendimento
- Visualização de tickets abertos
- Tratamento e acompanhamento de chamados
- Exclusão de tickets em determinadas situações


# Aprendizados desenvolvidos no projeto

Durante o desenvolvimento deste projeto, foram trabalhados conhecimentos importantes como:

- Estruturação de páginas web;
- Estilização com CSS;
- Interações com JavaScript;
- Lógica de funcionamento com PHP;
- Aeparação entre front-end e back-end básico;
- Organização de arquivos em um projeto web;
- Construção de fluxo de atendimento semelhante a um sistema real.




**Possíveis melhorias futuras**

- integração com banco de dados real
- autenticação mais robusta
- separação maior entre área do cliente e área administrativa
- melhoria visual responsiva para diferentes dispositivos
- histórico completo de tickets e respostas
- filtros de pesquisa e relatórios no painel do colaborador.
